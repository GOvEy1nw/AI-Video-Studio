## Project purpose
AI Video Studio is a local-first desktop app for AI image, video, and audio generation. Forked from `deepbeepmeep/LTX-Desktop-WanGP`, it is being reshaped from an LTX-model-centric video editor into a Freepik/Higgsfield-style creative studio powered exclusively by WanGP. Free and community-focused, not commercial.

## Decisions
- Project identity: AI Video Studio, a local-first desktop app for AI image, video, and audio generation. Forked from deepbeepmeep/LTX-Desktop-WanGP (which itself is a fork of Lightricks/LTX-Desktop). The app is being reshaped from a LTX-model-centric video editor into a Freepik/Higgsfield-style creative studio powered exclusively by WanGP. Initial product focus: QuickGen image generation. Later: QuickGen video, audio/TTS, then Production workflows. Free/community-focused, not commercial.
- WanGP-only generation mandate: All normal product generation must route through WanGP. No cloud/API generation paths (LTX API, fal.ai/ZIT, Gemini) should be exposed as normal features. If a model cannot run through WanGP, it must not be exposed as a normal QuickGen model. Desired-but-unsupported models are tracked separately, not integrated via custom direct pipelines.
- Local-first product principle: The default product must not require cloud API keys. Users should be able to run the app locally, configure local runtime/models, and generate locally. Prompt/media should not be sent to external generation services for normal operation.
- Inheritance rule — preserve inherited systems by default: If LTX-Desktop-WanGP already has a working system for a feature (project management, gallery, generation cards, metadata, history, backend architecture, frontend routing, editor), do NOT redesign or rebuild it unless required by the explicit goals of the PRD. Prefer extension over replacement. The inherited implementation remains the source of truth.
- Remove, disable, or hide inherited features that depend on external generation providers: LTX API-only generation paths, cloud generation provider flows, API key onboarding, fal.ai / Z-Image API usage, Gemini prompt suggestion API usage, any feature that sends prompts/media to external services, direct non-WanGP model-loading paths, cloud text encoding, API-only macOS assumptions, telemetry (unless fully removed or strictly opt-in and transparent), billing/cost/API quota copy, LTX Desktop branding/copy.
- Curated model profiles: Do not expose every WanGP-supported model automatically. The main UI should show a curated set of tested model profiles. A model profile answers: display name, media type, WanGP model/type/settings mapping, supported inputs, LoRA support status, resolution availability, aspect ratio availability, install/availability status, experimental/stable status. Add a lightweight curated model profile layer only if the inherited app does not provide an equivalent clean mechanism.
- QuickGen-first development order: Images first, then Video, then Audio/TTS, and finally Production. QuickGen should be the fast creative loop (Prompt → Generate → Compare → Reuse → Vary → Save). QuickGen controls: media type selector, model selector, aspect ratio, resolution, prompt, reference input where supported, LoRA selector where supported, seed lock, generate/cancel/progress. Advanced settings may exist but should not dominate the main experience.
- Production tab deferred: Production (structured workflow for finished AI video projects) should be visible but disabled or marked as coming later — not built before QuickGen is stable. It should eventually support: idea/treatment, script, visual style, characters, locations, props, shot list, storyboards, per-shot prompts, image/video/audio generation, review/approval, export/editor handoff.
- LoRA support is a key differentiator for the local-first advantage. MVP scope: LoRA folder detection/configuration, compatible LoRA listing, manual LoRA import/copy, enable/disable per generation, strength slider where WanGP supports it, LoRA info saved with generation metadata. LoRA should appear as an optional QuickGen control (e.g. 'LoRA: None / Select...'), hidden/disabled for models where LoRA is not available.
- Video editor is a bonus, not the MVP centre. Keep the inherited editor code if it does not slow development. Hide it, beta-label it, or leave it as-is. Do not rewrite it for MVP. Future use: review area for generated clips, simple assembly space, variation comparison, bridge from QuickGen into Production, rough AI animatic tool.
- MVP 0.1 — Local-Only QuickGen Image Foundation: Fork LTX-Desktop-WanGP into AI Video Studio. Keep inherited project/generation/gallery foundations. Remove/disable visible cloud/API usage. Generate images locally through WanGP only. Support at least one stable WanGP image model end-to-end. Introduce simple curated WanGP model addition. Keep output/history aligned with inherited app. Keep Production visible but disabled or marked as coming later.

- Phase 4 friendly availability states the UI must distinguish: Available, Missing model files, Partially installed, Unsupported by current WanGP install, Experimental, Hidden.
