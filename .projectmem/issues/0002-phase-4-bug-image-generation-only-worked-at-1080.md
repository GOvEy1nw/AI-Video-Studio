# #0002 Phase 4 bug: image generation only worked at 1080p (Z-Image) or 1080p/1440p (Krea 2). 540p and 720p failed with 'Unsupported image resolution mapping'. Root cause: frontend/hooks/use-generation.ts generateImage unconditionally called getImageDimensions(settings) which throws when settings.imageResolution is not in IMAGE_SHORT_SIDE_BY_RESOLUTION (a legacy map containing only 1080p/1440p/2048p). The new profile-driven path doesn't need getImageDimensions at all — the backend resolves exact WxH from (profileId, tier, aspect) — but the legacy call ran first and threw before the profile-based request body was built. Fix: only call getImageDimensions when settings.imageProfileId is falsy (backwards-compatible raw width/height path). When a profile is set, skip getImageDimensions entirely and send modelProfileId/aspectRatio/resolutionTier in the request body; the backend's _resolve_profile_and_dimensions resolves the exact WxH from the curated _RESOLUTION_TABLE. TypeScript clean. No backend changes needed.
