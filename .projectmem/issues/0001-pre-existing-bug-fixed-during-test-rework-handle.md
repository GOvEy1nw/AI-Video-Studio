# #0001 Pre-existing bug fixed during test rework: handlers/video_generation_handler.py:504 and handlers/image_generation_handler.py:80 — the _generate_via_wangp methods had a broad `except Exception as e: ... raise HTTPError(500, str(e)) from e` that caught validation HTTPError(400)s (Image file not found, Invalid image file) and re-wrapped them as 500 Internal Server Error. Fix: added `except HTTPError: self._generation.fail_generation(e.detail); raise` before the generic except so intentional client-error responses (400, 409) propagate unchanged. Only genuine internal errors get wrapped as 500. The test test_i2v_nonexistent_image, test_i2v_rejects_invalid_image_content_400, and test_http_400_logs_without_traceback now correctly see 400 responses.
